## How did we get here? * Learn about the history of configuration in Drupal * the rise and fall of the Features module * the promise of a better way * and the ultimate decision to develop a new configuration system for Drupal 8.
## Why is it important? * real world examples * explain why the configuration system is so important * cut your development time by as much as 30% or more * a virtually fail safe solution for deploying Drupal * one-person show * or a large distributed team deploying mission critical websites.
## How does it work? * basic fundamentals of configuration management * difference between “active” configuration and configuration in “code” * and why it matters. * basics of the Configuration API * the configuration UI * critical Drush commands to effectively manage configuration
## Okay, but how does it REALLY work? * Managing configuration in a team environment -- learn how to do fail-safe deploys no matter who is working on a ite * Configuration Split module -- learn how to use different configuration schemes depending on your environment * Configuration exclusions -- learn how clients can make configuration changes on things like Webforms without losing work * Gotchas -- learn what to do when things go crazy and the key_value_expire table takes over your configuration * How to recover when things go bad -- avoid configuration loss and overriding work
