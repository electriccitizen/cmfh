How is it "managed"?

  • Exporting your "active" configuration to code
  • Moving into your defined "sync" directory
  • It is now known as your "staged" config
  • The first time you do this you are in a 1:1 state
  • Active config matches staged config EXACTLY
  • Your "staged" config is now managed in code
  • Hopefully it is in git -- it has revisions, history
