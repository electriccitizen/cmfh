ELI5

Explain it like I'm five
  • Before I show some working demos I want to attempt a high level overview of what is really happening behind the scenes
  • Don't worry if it isn't entirely clear after this...
  • On day zero there is no "configuration management"
  • Everything you add or build is stored in your database
  • It is your "active" configuration and is what all of your users will see when they visit the site
  • To "activate" configuration management your first step is export your "active" configuration to code
  • The first time you do this, everything is in a 1:1 state
  • Your "active" config in the database matches your exported config EXACTLY
  • Your configuration is now in code, and now your team can "import" this configuration to their own local machines
  • Now they have a local copy of the site that also matches the production copy exactly
  • This where the fun stuff happens...the whole point of this is to be able to make and push changes or new features
  • Starting on local copy of the site that matches the production site is the first step to make this happen
  • Once you start making changes or add new features, YOUR active configuration no longer matches the active configuration on production
  • So your next step is to EXPORT your new active configuration back to code
  • Now that your configuration is exported to code, you can IMPORT it to any environment
  • After import, it has replaced the old configuration is now your "active" configuration
  • Assuming this is a staging server, your clients can now log on to see all of the great changes you have made
  • Assuming everybody is happy with the changes, you can now IMPORT them again to your production server
  • Now your new changes are part of the "active" configuration and your users will also be able to see your changes
  • That was a high level and kind of conceptual overview -- if you are still confused it is okay
